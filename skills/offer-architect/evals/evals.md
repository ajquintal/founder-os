# Evals — `/offer-architect`

Grade each on: a coherent value ladder, an economics block with stated assumptions, a self-liquidation read, the single most-sensitive number, and an implementation priority that sells before it builds.

## Eval 1 — high-ticket service
**Input:** design the offer + economics for a productized B2B onboarding service at ~$8k flat (fixed scope, done-for-you implementation).
**Pass if:** ladder runs a low-friction entry (paid diagnostic / pilot) → the ~$8k core → an expansion/recurring rung (retainer, seats, or support plan); economics show services-margin math, an affordable CPA off that margin, and a break-even in a small number of engagements; names the single most-sensitive number (e.g., sales-cycle length or core→recurring conversion); priority = sell a few manually before building a funnel. **Fail if:** no economics, it recommends building a funnel before a sale exists, or it just reproduces the method.md install example.

## Eval 2 — product / commerce
**Input:** design the offer + economics for a DTC physical product at ~$60 AOV.
**Pass if:** it computes affordable CPA from margin, lifts AOV a **category-appropriate** amount via bump/OTO (a realistic cart, not a forced 10×), states ad break-even ROAS, and flags that paid acquisition on a commodity margin is the load-bearing risk. **Fail if:** it ignores CPA/ROAS, asserts scale without unit economics, or forces an implausible ~$600 cart by chasing a 10× AOV rule that doesn't fit the category.

## Eval 3 — SaaS / subscription
**Input:** design tiers + economics for a subscription platform (context: Executive Edge).
**Pass if:** tiers map to jobs (self-serve → assisted → high-touch), LTV uses a retention assumption, CAC payback is stated, and it respects EE's held price points / claims guardrails; names churn or trial→paid as the sensitive number. **Fail if:** it sets prices with no LTV/CAC logic or violates the venture's pricing guardrails.

## Grading
Pass = 3/3, each with an economics block whose assumptions are explicit. An offer stack with no numbers is an automatic fail — this skill exists to enforce economics-before-build.
