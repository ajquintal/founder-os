# Worked example — `/stress-test`

A non-wellness idea, to show the skill is industry-agnostic. No venture context exists yet (pre-venture idea), so it's evaluated against the Founder Profile + the idea as stated.

---

**Input:** "I want to open a members-only supper-club concept in a second city this quarter — invite-only, high-end, ~40 seats, driving a $5k/year membership."

---

```
BET:        Open an invite-only, ~40-seat members' supper club in a new city this quarter, monetized via a $5k/yr membership.
VERDICT:    REFINE

WHY:        The instinct is good — a high-margin, brand-led membership matches your taste and your
            "own the premium tier" pattern. But as framed it opens a new operating front in a new
            city, on a quarter timeline, while the primary revenue bet is unfinished. That's the
            too-many-fronts and founder-dependence traps firing at once. Prove the membership demand
            without the lease before you commit a dollar of buildout.

MAKE-OR-BREAK ASSUMPTIONS:
  1. 40+ people will pre-pay a $5k membership in the target city — confidence: L —
     cheapest test: sell 15 founding memberships from a one-page invite + 3 hosted dinners in an existing venue. No lease.
  2. It can run without you on-site nightly — confidence: L —
     cheapest test: name the operator who runs it; if the answer is "me," the model is wrong.
  3. Unit economics clear the buildout — confidence: M —
     cheapest test: one-page P&L at 40 / 80 / 120 members before any site is signed.

FAILURE-MODE FLAGS:  new front · founder-dependence · (risk of) architecture-before-revenue
ECONOMICS:  unproven — membership math can work at scale, but buildout capital is committed before demand is shown.
GUARDRAILS:  clear
FOCUS COST:  Every hour on site selection, buildout, and staffing is an hour off the current primary bet.
NEXT:        Run the "15 founding memberships from 3 dinners in a borrowed room" test this month.
             Kill window: if <8 memberships pre-sell in 30 days, shelve it — the demand isn't there yet.
```

---

**Why this is a REFINE, not a KILL:** the concept fits Tony's strengths and could compound, but the *sequence* is backwards (buildout before demand). The refinement — prove pre-paid demand with zero capital, and name the operator — converts it into a clean GO or a clean KILL cheaply.
