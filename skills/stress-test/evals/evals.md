# Evals — `/stress-test`

Three cases spanning the verdict range. A correct run must produce the expected verdict AND the required elements. Grade pass/fail on both.

---

## Eval 1 — a strong bet → GO
**Context:** Executive Edge (`ventures/executive-edge`).
**Input:** "Productize the Executive Edge B2B cohort as a fixed $30k / 6-month offer and take it to warm corporate contacts."

**Expected verdict:** GO.
**A correct run must:**
- Recognize it compounds the *live* revenue engine rather than opening a new front.
- Flag economics as plausible (high-ticket, low volume needed).
- Name make-or-break assumptions like "warm buyers convert" and "cohort delivers without Tony nightly," each with a cheap test (e.g., book 3 discovery calls before building collateral).
- Fire the **founder-dependence** trap on delivery and require it **managed** for the GO to stand — a named plan to take Tony out of nightly delivery (SOP / handoff / hire), not a trap merely noted and waved through.
- Guardrails clear **only if** contacts are non-LYV; if any contact is LYV-sourced, it must flag the firewall.
- Give a single next action + kill window.
**Fail if:** it kills/refines a bet that compounds the primary engine with warm demand, ignores the LYV firewall check, or lets the GO stand while leaving the fired founder-dependence trap unmanaged.

---

## Eval 2 — a shiny distraction → REFINE or KILL
**Context:** Executive Edge (or none).
**Input:** "Launch a DTC longevity supplement brand alongside the platform."

**Expected verdict:** REFINE or KILL.
**A correct run must:**
- Fire **new front**, **founder-dependence**, and likely **architecture-before-revenue**.
- Attack the economics of a commodity category with no wedge and paid-acquisition dependence.
- State the focus cost against the current sprint explicitly.
- If REFINE, the only acceptable refinement is a near-zero-cost demand test (e.g., sell a curated stack to existing members before manufacturing anything).
**Fail if:** it returns GO, or fails to name the focus cost / new-front trap.

---

## Eval 3 — a fatal guardrail breach → KILL
**Context:** Executive Edge (`ventures/executive-edge`).
**Input:** "Sell Executive Edge operator licenses to med-spas in LYV's home market, sourced from LYV contacts."

**Expected verdict:** KILL.
**A correct run must:**
- Trigger the **guardrail check**: LYV firewall + competitor conflict in a co-owned venture's market (reputation/fiduciary risk).
- Set `GUARDRAILS: CONFLICT` and KILL regardless of any commercial upside.
- Not get seduced by the revenue math into a REFINE.
**Fail if:** it returns anything other than KILL, or treats the conflict as a manageable risk rather than a hard stop.

---

## Grading
Run all three. **Pass = 3/3 correct verdicts with required elements.** A wrong verdict on Eval 3 (guardrail) is a critical failure even if 1 and 2 pass — the guardrail override is non-negotiable.
