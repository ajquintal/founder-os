# /new-venture — Intake method

One level deep. Goal: a complete venture context in minutes, with the fewest questions.

## Inference rules (ask less)

- **Never ask what the Profile answers.** Voice, standards, delegation rules, guardrails, and Tony's time/budget/risk constraints are inherited silently — state that they're carried in, don't re-ask.
- **Draft, don't interrogate.** From whatever Tony says, pre-fill every template field with a best guess, then ask him only to correct the guesses. "Here's what I inferred — fix anything wrong" beats a blank questionnaire.
- **Batch the real unknowns** into one pass, grouped. Target ≤10 questions; drop any already answered.

## The unknowns worth asking (only if not inferable)

1. Name / working codename.
2. One-line thesis (what it does → the outcome).
3. Who exactly pays, and for what pain.
4. Revenue model + any known price points.
5. Ownership (sole / co-owned — triggers firewall rules) + regulatory surface.
6. Stage + the goal/deadline + what would make you kill it.
7. Assets already in hand (domain, entity, audience, partners, capital).
8. The wedge — the one thing no competitor ships that this can own (or "unknown → run /market-scan").

## Write + confirm

1. Slugify the name (kebab-case) → `ventures/<slug>/venture-context.md` from the template.
2. Fill every field; mark inferred-but-unconfirmed items so they're visible.
3. Emit the **approval card** (one screen): name · thesis · ICP · model · wedge · stage · goal + kill criteria · the 3 riskiest unknowns.
4. On Tony's yes/adjust, finalize. Then recommend the next skill: `/stress-test` (raw idea) or `/market-scan` (category not yet mapped) or `/offer-architect` (offer/pricing next).

## Guardrails carried into every new venture

Drafts-only outbound · LYV firewall (if any co-ownership/relationship overlap) · no medical/legal claims without review · no spend without approval. These are defaults, not options.
