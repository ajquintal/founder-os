# /content-cadence ‚Äî Steady-State Content Rhythm

The ongoing rhythm that produces and ships content on schedule, without Tony in the daily loop.
Not a one-time plan ‚Äî a repeatable run-loop over the system `/content-engine` stood up: recurring
calendar + batch workflow + repurpose loop. Every piece ladders toward the `/offer-architect` rungs
and repeats the wedge from `/positioning` (or `/market-scan`).

## Load first
1. `founder-profile/PROFILE.md` ‚Äî failure modes (overbuild, too-many-fronts, founder-dependence).
2. `founder-profile/voice-and-brand.md` ‚Äî the voice every piece inherits + banned words.
3. `founder-profile/operating-playbooks.md` ‚Äî GTM (#4: owned audience, lead with the wedge), compound assets (#6).
4. `founder-profile/delegation-protocol.md` ‚Äî drafts-only, batch the report not the permission.
5. `founder-profile/guardrails.md` ‚Äî drafts-only publishing, no medical/legal claims, sourcing firewall.
6. Active `ventures/<slug>/venture-context.md` ‚Äî channels, ICP, the wedge, brand deltas, cadence constraints.
7. Sibling skills, if present: `/content-engine` ‚Äî the system this rhythm runs; `/offer-architect` ‚Äî the ladder/rungs every piece ladders toward; `/positioning` (or `/market-scan`) ‚Äî the wedge each piece repeats.

## Method (full detail + worked example in `references/method.md`)
1. Anchor to the wedge + ICP: every piece ladders back to the one differentiated thing.
2. Pick the minimum viable channel set ‚Äî the 1‚Äì2 channels the ICP actually uses. Resist sprawl.
3. Set the recurring calendar: fixed weekly slots by format + a monthly theme cycle, sustainable on a cheap model.
4. Define the batch workflow: produce a full cycle in one sitting ‚Üí one Tony approval ‚Üí schedule/queue.
5. Build the repurpose loop: one anchor asset ‚Üí atomize into many derivatives ‚Üí recycle winners (evergreen bank).
6. Govern by the outcome metric (leads/calls/revenue), not reach; kill formats that don't move it.
7. Guardrail pass: drafts until approved; no medical/legal claims; source within the venture firewall.

## Rules
- Steady-state engine, not a one-time plan ‚Äî the deliverable is a rhythm that runs without Tony.
- Minimum viable channel set. A new channel is a new front ‚Äî justify it against the ICP or cut it.
- One anchor in, many pieces out. Never single-use content; every asset is built to atomize and recycle (compound).
- One approval gate per cycle, not per post. Batch the review; Tony's time is ‚â§15 min on approvals.
- Every piece ladders to the wedge and inherits the venture voice; banned words apply; no medical/legal claims.
- Govern by the outcome metric, not reach. Kill formats that don't produce it within their window.
