# /content-cadence — Steady-State Content Rhythm

The ongoing rhythm that produces and ships content on schedule, without Tony in the daily loop.
Not a one-time plan — a repeatable run-loop over the system `/content-engine` stood up: recurring
calendar + batch workflow + repurpose loop. Every piece ladders toward the `/offer-architect` rungs
and repeats the wedge from `/positioning` (or `/market-scan`).

## Load first
1. `founder-profile/PROFILE.md` — failure modes (overbuild, too-many-fronts, founder-dependence).
2. `founder-profile/voice-and-brand.md` — the voice every piece inherits + banned words.
3. `founder-profile/operating-playbooks.md` — GTM (#4: owned audience, lead with the wedge), compound assets (#6).
4. `founder-profile/delegation-protocol.md` — drafts-only, batch the report not the permission.
5. `founder-profile/guardrails.md` — drafts-only publishing, no medical/legal claims, sourcing firewall.
6. Active `ventures/<slug>/venture-context.md` — channels, ICP, the wedge, brand deltas, cadence constraints.
7. Sibling skills, if present: `/content-engine` — the system this rhythm runs; `/offer-architect` — the ladder/rungs every piece ladders toward; `/positioning` (or `/market-scan`) — the wedge each piece repeats.

## Method (full detail + worked example in `references/method.md`)
1. Anchor to the wedge + ICP: every piece ladders back to the one differentiated thing.
2. Pick the minimum viable channel set — the 1–2 channels the ICP actually uses. Resist sprawl.
3. Set the recurring calendar: fixed weekly slots by format + a monthly theme cycle, sustainable on a cheap model.
4. Define the batch workflow: produce a full cycle in one sitting → one Tony approval → schedule/queue.
5. Build the repurpose loop: one anchor asset → atomize into many derivatives → recycle winners (evergreen bank).
6. Govern by the outcome metric for the venture's model (orders/purchases, leads, booked calls, bookings, revenue), not reach; kill formats that don't move it.
7. Guardrail pass: drafts until approved; only substantiated claims, any regulated claim (health/financial/legal/environmental) routed for review per the venture's claims surface; source within the venture firewall.

## Output contract — three concrete artifacts

```
  Channels:      <1–2 chosen + the role of each>
  Weekly slots:  <Mon format · Wed format · Fri format …>
  Monthly cycle: <Wk1 theme · Wk2 · Wk3 · Wk4>
  Approval gate: <when Tony reviews the batch — one review per cycle, ≤15 min>
BATCH WORKFLOW (one cycle, one sitting)
  1. Ideate against the calendar (cheap model)    2. Draft all pieces
  3. Tony approves the batch (≤15 min)            4. Schedule/queue to dates
REPURPOSE LOOP
  Anchor:   <one long-form asset per cycle>
  Atomize:  <derivative formats mapped to calendar slots>
  Recycle:  <evergreen-bank rotation of top performers>
  Governing metric: <the outcome metric that keeps or kills a format>
```

## Rules
- Steady-state engine, not a one-time plan — the deliverable is a rhythm that runs without Tony.
- Minimum viable channel set. A new channel is a new front — justify it against the ICP or cut it.
- One anchor in, many pieces out. Never single-use content; every asset is built to atomize and recycle (compound).
- One approval gate per cycle, not per post. Batch the review; Tony's time is ≤15 min on approvals.
- Every piece ladders to the wedge and inherits the venture voice; banned words apply; make only substantiated claims and route any regulated claim (health / financial / legal / environmental) for professional review before publish — honor the venture's own claims surface (`venture-context` + `engineering-backbone §9`), not a fixed medical/legal-only list.
- Govern by the outcome metric, not reach. Kill formats that don't produce it within their window.
