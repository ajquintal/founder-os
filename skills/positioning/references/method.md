# /positioning — Full method & worked example

One level deep; does not fan out further. Lens: April Dunford (*Obviously Awesome*) — positioning is a deliberate choice made *relative to the alternatives*, not a slogan you assert.

## The five components (Dunford), in build order

1. **Competitive alternatives.** What does the buyer use if you don't exist? Include the real defaults — a spreadsheet, an incumbent, an agency, or *nothing / willpower*. This is the reference frame; everything else is relative to it.
2. **Unique attributes.** The features/capabilities you have that those alternatives lack — pulled straight from the market-scan wedge. Concrete ("closed-loop protocol scoring"), not vague ("better UX").
3. **Value (+ proof).** The benefit each attribute *enables* that a buyer cares about. Chain it: attribute → value → proof. If you can't state the value or prove it, the attribute doesn't belong in the positioning.
4. **Target segment.** The buyers who care *most* about that value, plus the observable traits that let you spot them. Positioning is FOR them; a segment lukewarm on your value is the wrong segment.
5. **Market category.** The context you place the product in so the value is *obvious* to that segment — a frame you choose, not the default label everyone already uses. (+ optional **trend**: a shift that makes the value urgent now — use sparingly; never lead with a trend over value.)

## Messaging pillars

The 3–5 repeatable themes every asset (page, email, ad, deck) hangs on. Each pillar = a value + its proof + the objection it removes. Pillars are the bridge from positioning to `/direct-response-copy`: copy *dramatizes* the pillars; it does not invent new claims.

| # | Pillar | The claim (one line) | Proof / why believable | Objection it answers |

## Worked example (abridged) — Executive Edge

- **Alternatives:** blood-only lab platforms (Function $365/yr, Superpower $199/yr, Lifeforce $129/mo); device-only trackers (Whoop $30/mo); TRT clinics (Marek $250/mo); experts/content (Attia); and *do-nothing / willpower*.
- **Unique attributes:** the closed loop (assign protocol → score execution → detect drift → adjust); blood + continuous-device fusion; daily execution scoring (Daily Edge Score, Drift, BRI).
- **Value + proof:** you don't just *see* your data — the system runs your protocol and corrects it daily (value), proven by the scoring IP live on the user base and the Junction + Oura/Withings integrations (proof).
- **Target segment:** high-performing men ~35–55 who want a system they *operate*; fear coded as performance ("I'm not performing at my level"), not aging. Spot them: owns a wearable, has run a panel, high-agency.
- **Category frame:** "Executive Performance OS — the execution layer." Deliberately NOT "another labs-and-insights app"; that frame makes the daily-execution value obvious and steps out of the commoditized lab category.

**Positioning statement:**
```
CATEGORY:   Executive Performance OS (the execution layer)
FOR:        high-performing men 35–55 who refuse to decline and want a system they operate
WHO:        have the data (labs, a wearable) but don't know what to do daily — or whether it's working
EXECUTIVE EDGE IS THE performance OS THAT runs your protocol, scores how you execute it daily, detects drift, and adjusts.
UNLIKE:     labs-and-insights platforms (Function, Superpower, Lifeforce) that hand you a snapshot, trackers (Whoop) that watch without directing, or do-nothing / willpower
BECAUSE:    it closes the loop and fuses blood + continuous device data into one readout
PROOF:      Daily Edge Score / Drift / BRI live on the user base; Junction panels + Oura/Withings fused
TREND:      personalized-health AI heavily funded; wellness-tech ~$65B (2026) → ~$208B (2035)
```
